1 Number Length Directed Graph

1.1 Problem

In his YouTube video Four has Four Letters, Matt Parker creates cyclic directed graphs out of counting systems. Given a language's counting system, count the number of letters in the spelled-out representation of a number, and then go to the number that is the character count. For any language, this forms a directed graph that ends in a cycle or cycles with one or more numbers in the cycle. He poses a number of challenges in the video.

  1. Find longer chains in English.
  2. Find a language with a bigger loop than French.
  3. Find the k threshold for a language. All numbers bigger than k have spelling lengths less than the number itself.

Parts of the problem are solvable in SQL using the recursive queries; specifically, transitive closure, which I covered in my previous blog post.

1.2 Tools

To create a table of numbers and their spelling lengths, I decided to use the Emacs package, spell-number.el. The spell-number package provides the function spelln-integer-in-words, which given an integer, will give the spelled representation. The particular language can be selected by setting the spelln-language variable. The default is portuguese-br, Brazilan Portuguese.

The emacsql.el package was used to insert data into an SQLite database. Directed graphs were made using Graphviz and ImageMagick. This document was made using Emacs Org mode.

1.3 Data Representation

The spell-number package supports Catalan, Danish, Dutch, Esperanto, Finnish, German, Italian, Japanese (Romanji), Norwegian, Spanish, Swedish, three English dialects, two French dialects, and two Portuguese dialects.

The British dialect of English uses the traditional European counting system, https://en.wikipedia.org/wiki/Names_of_large_numbers; milliard and billiard. The European English version is ported from the Spell::Number Perl package. It uses the Long Scale (Traditional British). The European English version leaves off the number one in numbers such as 1001. The US English version uses the Short Scale. All English versions use `and'; although, not consistently. 101 is one hundred one in US English, but 1001 is one thousand and one. The British dialect always uses "and". In the European dialect, the trailing one is left off in numbers such as 1001, so "one thousand and". The US and European dialects only start differing from 1001; numbers below this are spelled the same. British English starts differing at 101, due to the inclusion of "and". There does not appear to be an option to remove `and', except for in currency spellings, spelln-and-p. I found one error in spell-number.el; specifically, quadrillion is mis-spelled as quatrillion. Hopefully, this is its only error.

The French dialects are French from France (French_fr) and non-France versions (French_ch), such as what is spoken in Switzerland, apparently. The Portuguese dialects are those spoken in Portugal (Portuguese_pt) and Brazil (Portuguese_br) see https://en.wikipedia.org/wiki/Portuguese_orthography#cite_ref-3.

The largest value that spell-number can "say" is the largest hundred-quadrillion or 999,999,999,999,999,999, which is within Emacs's numeric range. Dutch and Finnish only go up to 999,999,999. On my system, Emacs's max integer, most-positive-fixnum, is 2,305,843,009,213,693,951, so spelln-integer-in-words will work for any value that spell-number can spell.

Catalan, Esperanto, Finnish, French-ch, French-fr, German, Italian, Japanese, Norwegian, Portuguese-Br, Portuguese-Pt, Spanish, and Swedish contain non-ascii characters. The Japanese version is Romanji. All characters were counted as single characters. Spaces, commas, and dashes were removed for counting.

Only the natural numbers 0 to 1 million were counted for each language, and only their numeric value and spelled character lengths were stored in the SQL database.

1.3.1 Dialect Differences

The following queries use a Common Table Expression (CTE), named diff, that does a set difference between two language tables, returning those entries in the first table that are not in the second. Only the first differing value is returned. The second table is joined with the diff table and the numbers where the two tables differ plus their respective lengths are displayed.

    WITH diff AS (
     SELECT number, length AS length_fr
     FROM
     (SELECT * FROM French_fr
     EXCEPT
     SELECT * FROM French_ch)
     ORDER BY 1 ASC LIMIT 1

SELECT fr.number AS number, length_fr, length_ch
FROM

    (SELECT number FROM diff)) AS fr

1.4 Populating the Database

I created number-spelling-length.el to find the length of numbers in all languages supported by spell-number. It can be run as a batch script or interactively. To run as a batch script, it is best to compile the code using the byte-compile-file function. You may need to change the file paths for the required packages at the beginning of the code. The batch file needs to be in the same directory as the database and can be run using the command 

emacs -batch -l number-spelling-length.elc

number-spelling-length.el populates number-spelling-length.db tables with number, length pairs. The numbers' spelled representations are not stored. The file also includes the function nsl-list-number-language-length, which you can use to get the length and spelled version of a number for different languages. 

(nsl-list-number-language-length 30001 'english-us 'english-gb 'english-eur)

Also included is the function, nsl-list-numbers-in-language, which lists spelled representations and their lengths for given numbers, in the given language. Both nsl-list-numbers-in-language and nsl-list-number-language-length return a list of lists, which can be converted into a table by Org mode. 

(nsl-list-numbers-in-language 'english-us 1 2 3 4 10 20)

The nsl-number-length function gives the length of a number in the given language. It is referenced by all of the other functions. 

(nsl-number-length 'english-us 10)

The nsl-create-language-data function applies nsl-number-length to a range of values and returns a list of number, length vectors that can be used by emacsql. 

(nsl-create-language-data 'english-us 10 1)

The nsl-insert-language-data function calls the nsl-create-language-data function and inserts the results into the SQL database. It does it in chunks of 500 inserts per SQL call with emacsql, since longer lists caused errors. It uses the insert-language-data-500 private helper function to perform the inserts.

The batch program iterates over a list of (spelln language symbols . table name) cons cells and calls the nsl-insert-language-data function for each pair.

1.6 Loops

The language with the largest number in a loop is Finnish, with a loop between 8 and 9, or kahdeksan and yhdeksän. Danish, Norwegian, and Esperanto each have three independent loops. French has the largest loop.

1.6.1 Finding Loops

The transitive closure can be used to detect loops. Any number that is part of a loop, will loop directly onto itself in the transitive closure. 

WITH RECURSIVE
nodes(number, length) AS (
    SELECT number, length FROM $in_table WHERE number <= (SELECT MAX(length) FROM $in_table)
),
trans_closure(itter, number, length) AS (
    SELECT 0, number, length FROM nodes
    UNION ALL
    SELECT itter + 1, A.number, B.length
    FROM trans_closure AS A JOIN nodes AS B
    ON A.length = B.number
    WHERE itter < (SELECT (SELECT COUNT(*) FROM nodes) * (SELECT COUNT(*) FROM nodes))
    ORDER BY 1 ASC
)
SELECT DISTINCT number FROM trans_closure WHERE number = length;

1.7 Smallest Number of a Given Length

The smallest_number query returns the smallest number having a particular length, ignoring zero, and for numbers greater than or equal to 100 and 100,000. Zero is added in for completeness, but "zero" is not used in naming larger numbers in English. These numbers are used to find the likely next numbers in the English chains. 

SELECT num AS number, length FROM
(SELECT num, length FROM
   (SELECT num, length FROM
      (SELECT MIN(number) AS num, length FROM $in_table WHERE number > 0 GROUP BY length
      UNION ALL
      SELECT number AS num, length FROM $in_table WHERE number = 0))
   UNION ALL
   SELECT MIN(number) AS num, length FROM $in_table WHERE number >= 100 GROUP BY length)
UNION
SELECT MIN(number) AS num, length FROM $in_table WHERE number >= 100000 GROUP BY length
ORDER BY 2 ASC, 1 ASC;

1.8 Finding Chains

An SQLite query, which is detailed at the end of this section, was used to find the minimum starting value for every path length in each of the tables. Nodes that are part of a loop have a path length of zero. Esperanto, Norwegian, and Swedish have the shortest, maximum path lengths at three connections each. 1.8.1 English

The US and EUR English count the same until 1001, not using "and". While the British English version uses "and" but is otherwise the same. There are six possibilities for the next number; three numbers per period with/without "and", six numbers per period with/without "and", and the traditional European system with/without "and" with three numbers per period. The traditional scale is called long scale because a Billion means a Million, Million, and so on, https://en.wikipedia.org/wiki/Long_and_short_scales. However, the traditional scale includes Milliard, Billiard, etc, increasing by factors of 1000; and using six numbers per period makes it the same as the Traditional British scale.

US English and European English have the same graphs. These graphs represents numbers without using "and". I estimated the next numbers in the chains, having path length seven. More on the in the section Longest Chains English Variants. There is probably not a named number in English with a path length of eight.

1.8.16 Code

The paths starting with the smallest values are found using the following queries, including five common table expressions (CTEs). $in_table is an Org Babel variable that is set before the code block is sent to SQLite. It is set to a table name, such as English_us for example.

The nodes CTE returns a table where leaf nodes, nodes that do not have child nodes, are removed. It finds the largest non-leaf node and removes any larger nodes. Some leaves might still be present, but this function is only meant to prune extraneous information for the trans_closure function, so removing all of the leaves is not necessary.

The trans_closure CTE computes the transitive closure of the table returned by nodes. The resulting table is used by the non_loop_members CTE to remove nodes from the $in_table that are part of a loop. A node that is part of a loop will have a loop directly back to itself in the transitive closure. The non_loop_members table is used by the path_group CTE to compute paths starting from start.

The path_group CTE is similar to the itineraries CTE in my previous post. The "start" field is used to group tree traversals starting from a node. "number" is a node in the traversal. "length" is the next node in the traversal, which is also the spelled-out length of the current node number. Because non-loop members are removed, the table that path_group operates on is acyclic, so the recursive CTE is guaranteed to finish and no limit or counter is needed.

The smallest_path_lengths CTE takes path_group's output and groups path itineraries by their starting node, start. It counts the number of entries (x) in each group, returning the path length, p_length, and starting value for each group. Numbers with equal path lengths (x) are grouped together, and the smallest starting value out of each group is taken.

The final query joins the path_group table with the smallest_path_lengths table to get the path members from path_group for paths starting with the "start" values found by smallest_path_lengths. The node number and destination are formatted to make it easier to insert into Graphviz code.

This query required 30 seconds of running time on my computer. The majority of which was spent on the last two parts of the query.

-- .timer ON
WITH RECURSIVE
nodes(number, length) AS (
    SELECT number, length FROM $in_table WHERE number <= (SELECT MAX(length) FROM $in_table)
),
trans_closure(itter, number, length) AS (
    SELECT 0, number, length FROM nodes
    UNION ALL
    SELECT itter + 1, A.number, B.length
    FROM trans_closure AS A JOIN nodes AS B
    ON A.length = B.number
    WHERE itter < (SELECT (SELECT COUNT(*) FROM nodes) * (SELECT COUNT(*) FROM nodes))
    ORDER BY 1 ASC
),

-- remove root nodes
non_loop_members(start, number, length) AS (
    SELECT number, number, length FROM $in_table
    WHERE number NOT IN
    (SELECT number
     FROM trans_closure
     WHERE number = length)
),
path_group(start, number, length) AS (
    SELECT start, number, length
    FROM non_loop_members
    UNION ALL
    SELECT A.start, A.length, B.length
    FROM path_group AS A JOIN non_loop_members AS B
    ON A.length = B.number
    -- no loop count check is necessary because cyclic nodes were removed
),
smallest_path_lengths(start, p_length) AS (
   SELECT start, MIN(x) AS p_length
   FROM (SELECT start, COUNT(*) AS x FROM path_group GROUP BY start)
   GROUP BY x
)
SELECT smallest_path_lengths.start, p_length,
       CAST(number AS text) || " -> " || CAST(length AS text)
FROM smallest_path_lengths
JOIN path_group ON path_group.start = smallest_path_lengths.start;
